    • OUTCOME_OUTCOME_HPP

    • OUTCOME_RESULT_HPP

    • OUTCOME_TRY - If the outcome returned by expression m is not valued, propagate any failure by immediately returning that failure immediately, else set *v( to the unwrapped value

    • OUTCOME_TRY2

    • OUTCOME_TRYV - If the outcome returned by expression m is not valued, propagate any failure by immediately returning that failure state immediately

    • OUTCOME_TRYV2

    • OUTCOME_TRYX - If the outcome returned by expression m is not valued, propagate any failure by immediately returning that failure state immediately, else become the unwrapped value as an expression. This makes OUTCOME_TRYX(expr) an expression which can be used exactly like the try operator in other languages.

    • outcome_v2_156a3156

      • bad_outcome_access - Thrown when you try to access state in a outcome<T, EC, E> which isn’t present.

      • bad_result_access - Thrown when you try to access state in a result<R, S> which isn’t present.

      • in_place_type_t - Aliases std::in_place_type_t<T> if on C++ 17 or later, else defined locally.

      • no_error_type - Placeholder type to indicate there is no error type

      • no_exception_type - Placeholder type to indicate there is no exception type

      • no_payload_type - Placeholder type to indicate there is no payload type

      • no_value_type - Placeholder type to indicate there is no value type

      • operator!=(const result<T, U, V> &, const outcome<R, S, P, N> &)”) - True if the result is not equal to the outcome

      • operator==(const result<T, U, V> &, const outcome<R, S, P, N> &)”) - True if the result is equal to the outcome

      • outcome - Used to return from functions (i) a value and (a positive status and/or a payload) or (ii) no value and (a negative status and/or a payload). constexpr capable.

      • result - Used to return from functions (i) a value (ii) a value and a positive status or (iii) no value and a negative status. constexpr capable.

    • outcome_v2_156a3156::impl

      • outcome_exception_observers - The exception observers implementation of outcome<R, S, P>. Only appears separate due to standardese limitations.

      • outcome_exception_observers<Base, R, S, void, NoValuePolicy>

      • outcome_failure_observers - The failure observers implementation of outcome<R, S, P>. Only appears separate due to standardese limitations.

      • outcome_failure_observers<Base, R, std::error_code, std::exception_ptr, NoValuePolicy>

      • outcome_payload_observers - The payload observers implementation of outcome<R, S, P>. Only appears separate due to standardese limitations.

      • outcome_payload_observers<Base, R, S, void, NoValuePolicy>

      • result_error_observers - The error observers implementation of result<R, EC, NoValuePolicy>. Only appears separate due to standardese limitations.

      • result_error_observers<Base, void, NoValuePolicy>

      • result_final - The assembled implementation type of result<R, EC, NoValuePolicy>. Only appears separate due to standardese limitations.

      • result_storage - The base implementation type of result<R, EC, NoValuePolicy>. Only appears separate due to standardese limitations.

      • result_value_observers - The value observers implementation of result<R, EC, NoValuePolicy>. Only appears separate due to standardese limitations.

      • result_value_observers<Base, void, NoValuePolicy>

      • select_result_impl

    • outcome_v2_156a3156::policy

      • default_outcome_policy - Default outcome<R, S, P> policy selector.

      • default_result_policy - Default result<R, S> policy selector.

      • error_code_throw_as_system_error - Policy interpreting EC as a type implementing the std::error_code contract and any wide attempt to access the successful state throws the error_code wrapped into a std::system_error

      • error_code_throw_as_system_error_exception_rethrow - Policy interpreting S as a type implementing the std::error_code contract, E as a type implementing the std::exception_ptr contract, and any wide attempt to access the successful state throws the exception_ptr if available, then the error_code wrapped into a std::system_error.

      • exception_ptr_rethrow - Policy interpreting EC as a type implementing the std::exception_ptr contract and any wide attempt to access the successful state calls std::rethrow_exception().

      • terminate - Policy implementing any wide attempt to access the successful state as calling std::terminate

      • throw_directly - Policy interpreting EC as a type to be thrown directly during wide checks.

    • outcome_v2_156a3156::trait - Namespace for traits

      • is_exception_ptr - Trait for whether type P is to be considered a payload to an exception.
